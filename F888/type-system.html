2. Terminology

2.1. "Variables"

The term "variable", when used within this document, represents any form of data which has a type.

3. List of Types

3.1. Natural Numbers (‚Ñï)

This type will be referred to as ‚Ñï within this document, and as "Nat" within the syntax of F888.

3.2. Booleans (ùîπ)

Boolean types are types which specify that the object being typed has exactly two possible states. These two states may be redefined, however by default the two states are True and False.

This type will be referred to as ùîπ within this document, and as "Bool" within the syntax of F888

3.3. Unions (‚à™)

A Value must be of a union type if said value may take on multiple different types. The said value may only be of one type at any given time. A union type is immutable, meaning that the list of types that the value may be are fixed, and cannot be changed. An example of a union type would be a union named "A" which contains the subtypes ‚Ñï and ùîπ. The types stored within the union may not be changed. Given the previous union example, a variable named "e" which is of type "A" may be of either subtype ‚Ñï or ùîπ, but not both at the same time.

The union type will be referred to as ùïå within this document, and as "Union" within the syntax of F888.

3.4. Tuples (ùïã)

This type may use any other type as a subtype, allowing for the contents a tuple to contain data of any type. All of the values contained within any given tuple must be of the same type.

This type will be referred to as ùïã within this document. and as "List" within the syntax of F888.

3.5. Functions (Œª)

This data type may use any other type as a subtype, allowing for functions which return any type. It is important to note that all functions are anonymous by default. A variable may be assigned of a function type, which allows the function to be referred to by the name of the variable which it is assigned to. Functions may accept data of any type as an argument(s), and return any data of any type as an output.

A function may also accept a type as arguments. If this is the case, then the function must also output a type. The inverse also hold, that being that a function may output a type, in which case, that function must also accept a type(s) as its arguments. Functions which accept a type(s) as input or outputs a type are classified under the subtype of a Constructor.

Functions may only accept one argument. In order to simulate a function which accepts more than one argument, currying is used. It is important to note that the syntax of function declaration allows for multiple arguments. In the case of multiple arguments being specified, a function will be curried at runtime.

This type will be referred to as Œª within this document, and as "Func" within the syntax of F888.

3.6. Constructors (Œõ)

A constructor is any function which accepts a type as its input. If a function accepts a type as its input, then it also must output a type. The inverse is also true.

Constructors will be referred to as Œõ within this document, and as "Cons" within the syntax of F888.

4. Typing

4.1. Type System Strength

F888 is fully strongly typed. No variable may have its type changed or reinterpreted.

4.2. Static VS Dynamic Typing

F888 is statically typed. All types are specified at compile time, according to the syntax of the source code.

There is also a type checker, which verifies that the types that have been specified will not result in a type error. If a variable's type does not match its usage, a type error would occur.

5. Syntax

5.1. Tuples

Let Œ± be the name of tuple containing two natural numbers, and let the contents of Œ± be named x and y respectively. Given this defintion, the contents of tuple Œ± would be specified using the following syntax: 

Œ± [x, y] : List (Nat)

In the case of a tuple being used as a subtype, only the type of the tuple is used. For example, in the case of a function named Œ±, which accepts two natural numbers as its arguments and has a natural number as its output, the type of Œ± would be written as the following: 

Func (List (Nat)) -> Nat

Note that regardless of the amount of items in a tuple, only one type is specified. This type is applied to all the elements in the tuple.

5.2. Type Specification

The type of a variable must always be specified. Let Œ± be the name of a (non-function) variable, and let A be the type. Given this definition, the type of Œ± is specified using the following syntax: 

Œ± : A

If a variable is a function, then the type of both its output and its input(s) must be specified. Let Œ± be the name of function, and let Œ± accept two variables of type ‚Ñï, and let Œ± output a variable of type ‚Ñï. Let the name of the arguments of Œ± be x and y respectively. In this example, Œ± returns the sum of x and y. Given this definition, the type of Œ± is specified using the following syntax: 

Œ±.(x, y)  : Func (Nat, Nat) -> Nat
